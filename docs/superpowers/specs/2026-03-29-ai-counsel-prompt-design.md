## 1. 배경 및 목적

### 왜 이 작업을 하는가

whatsyoursaju.com은 무료 사주 분석이 완성된 상태(83개 테스트 통과). 수익화의 핵심 상품은 **AI 사주 상담**(월 6,900원 구독). Phase 1에서 결제 시스템을 만들기 **전에**, "팔 물건"의 품질이 돈을 받을 만한지 검증해야 한다.

### 성공 기준

- AI 응답이 "GPT한테 사주 물어본 것"과 명확히 차별됨
- 기준 테스트 케이스 5개 질문에서 "와, 이걸 어떻게 알지?" 반응 유발
- 자동 검증 스크립트로 품질 기준 통과 (개인화 점수, 근거 제시, 금지어 미포함)

### 스코프 경계

**포함**: 시스템 프롬프트, 사주 JSON 직렬화, `_lastAnalysisResult` 확장, 검증 스크립트
**미포함**: 가격표 UI 변경, 무료/유료 체험 UX, 공유 카드, GA 이벤트, /api 엔드포인트 → 모두 Phase 1

## 3. 시스템 프롬프트 설계

### 3.1 프롬프트 아키텍처

```
[시스템 프롬프트 ~1,500 토큰] — 안정적, 프롬프트 캐싱 대상
  역할 정의
  6단계 해석 프로토콜
  시나리오별 해석 가이드
  톤 규칙
  금지사항
  정전(canonical) 용어 목록

[유저 메시지 ~1,200 토큰] — 매 호출 변경
  사주 컨텍스트 JSON (압축 형식, ~800 토큰)
  Bad/Good 예시 쌍 (~200 토큰)
  사용자 질문 (~100-200 토큰)
```

**Bad/Good 예시를 유저 메시지에 배치하는 이유**: 시스템 프롬프트는 Anthropic API의 프롬프트 캐싱 대상. 예시를 유저 메시지로 빼면 프롬프트 튜닝 시 캐시 무효화 없이 반복 가능.

### 3.2 역할 정의

```
전통 명리학 30년 경력의 전문가이자 현대적 라이프 코치.
억부용신 + 조후용신 체계 기반. 격국은 참고 요소로 활용.
신살은 성격적 경향성의 보조 지표로만 사용.
```

**학파 입장 명시**: 억부용신 + 조후용신 기반, 격국 참고. 이를 명시해야 AI 답변의 일관성이 유지됨.

### 3.3 해석 프로토콜 6단계

| 단계 | 내용 | 설명 |
|------|------|------|
| 0 | 질문 맥락 파악 | "왜 이 질문을 하는가" — 같은 "이직" 질문도 맥락에 따라 봐야 할 요소가 다름 |
| 1 | 일간 파악 + 격국/구조 확인 | 사주의 기본 유형 판단 (AI가 JSON 데이터에서 추론) |
| 2 | 신강/신약 + 용신/기신 | 기계 판단 결과를 받되, 사주 전체 구조 보고 AI가 보정 가능 |
| 3 | 궁위별 합충형파해 매핑 | 년주=사회궁, 월주=사업궁, 일주=배우자궁, 시주=자녀궁. "어디서 충이 일어나느냐 = 어느 영역 갈등" |
| 4 | 현재 대운 위치 + 세운 교차 | "지금 어떤 에너지가 흐르는가" + 향후 3년 전망 |
| 5 | 처방 | 원인(사주 근거) + 방향 + 시기(최소 분기 단위) + 구체적 행동 1-2가지 |
| 6 | 향후 주의 시점 | 3-6개월 내 주의할 시점 1개 제시 (재방문 트리거) |

### 3.4 응답 원칙

- **첫 2문장 콜드 리딩**: 사주 데이터 기반으로 사용자의 현재 심리 상태를 짚는 것으로 시작
  - 예: "지금 직장에서 답답함을 느끼고 계시죠? 경금(庚金) 일간에 올해 병오(丙午)년 편관의 압박이 최고조인 해입니다."
- **사주 핑거프린트**: 응답 시작부에 1줄 요약 강제
  - 예: "庚金 신강 | 쌍편재 + 상관생재 | 현재 丁丑 대운(정관운)"
- **대조적 추론(contrastive reasoning)**: 주요 주장에 "만약 다른 구조였다면" 대비 1회 이상 포함
  - 예: "편재가 아닌 정재였다면 안정적 직장 유지를 권했겠지만, 당신은 쌍편재 구조이므로..."
- **원국 vs 운 구분**: "원래부터 이런 특성" vs "지금 이 시기의 영향"을 명확히 분리
- **시기 특정**: "올해 좋습니다"가 아닌 "2026년 3~5월이 핵심" 수준의 구체성
- **클리프행어 종료**: 마지막에 "참고로 님의 사주에서 한 가지 더 흥미로운 패턴이 보이는데요..." 식 후킹

### 3.5 시나리오별 해석 가이드

| 시나리오 | 핵심 요소 | 필수 포함 |
|----------|-----------|-----------|
| 이직/커리어 | 관성(편관/정관) + 식상의 대운/세운 흐름 | 최적 이직 시기, 적합 직종 방향 |
| 연애/결혼 | 일지(배우자궁) + 재성(남)/관성(여) + 도화살 | 만남 시기, 배우자상, 관계 패턴 |
| 재물/투자 | 재성 구조 + 식상생재 여부 + 대운 재성 시기 | 재물운 최고/최저 시기 |
| 건강 | 오행 편중 + 기신 강화 시기 | 주의 장기/계절, 예방 행동 |
| 성격/강점 | 일간 특성 + 격국 구조 + 특수구조 | MBTI 대비 사주 관점의 차별점 |

### 3.6 톤 규칙

- 기본: 친근한 존댓말 ("~하실 수 있어요", "~해보시면 좋겠어요")
- 진지한 주제(이직, 건강, 결혼): 전문적 톤으로 전환 ("~라 판단됩니다", "~에 유의하시길 권합니다")
- 전문 용어 사용 시 반드시 괄호 순화 ("편관(偏官, 외부 압박의 에너지)")
- 한국어 전용, 영어 혼용 금지
- 결정론 금지: "~할 수밖에 없다" → "~한 경향이 있으므로 의식적으로 보완하면..."

### 3.7 금지사항

- 의료/법률/투자에 대한 구체적 조언 (종목, 약물, 소송 방향 등)
- 운명론적 절망감 유발 ("올해는 안 좋습니다" → "성장의 기회이기도 합니다")
- 사주 맹신 유도
- 제공된 정전 용어 외의 신살/용어 사용 금지
- 무료 분석 결과와의 모순 (AI는 무료 분석을 보강하는 역할)

### 3.8 정전 용어 목록

시스템 프롬프트에 포함하여 AI가 이 범위 밖의 용어를 생성하지 않도록 제한:

- **천간**: 甲乙丙丁戊己庚辛壬癸
- **지지**: 子丑寅卯辰巳午未申酉戌亥
- **오행**: 목(木), 화(火), 토(土), 금(金), 수(水)
- **십신**: 비견, 겁재, 식신, 상관, 편재, 정재, 편관, 정관, 편인, 정인
- **12운성**: 장생, 목욕, 관대, 건록, 제왕, 쇠, 병, 사, 묘, 절, 태, 양
- **신살 (6종만)**: 도화살, 역마살, 화개살, 천을귀인, 천덕귀인, 월덕귀인
- **합충**: 육합, 삼합, 상충, 삼형, 자형, 파, 해

## 4. 사주 컨텍스트 직렬화

### 4.1 `_lastAnalysisResult` 확장

**현재** (라인 4203, 8개 필드):
```javascript
window._lastAnalysisResult = {
  name, dayP, yearP, monthP, hourP, pillars, elementCounts, yongsinInfo
};
```

**확장** (19개 필드):
```javascript
window._lastAnalysisResult = {
  // 기존 8개
  name, dayP, yearP, monthP, hourP, pillars, elementCounts, yongsinInfo,
  // 인적 정보
  gender,          // '남'/'여' (selectedGender)
  birthDate,       // 'YYYY-MM-DD' (solarYear-solarMonth-solarDay)
  birthHour,       // 0-23 (hourVal)
  // 분석 결과
  specialStructures,  // detectSpecialStructures() 반환값
  sinsalList,         // detectSinsal() 반환값
  hapChung,           // findHapChung() 반환값
  daeunList,          // getDaeun() 반환값
  yearFortune,        // getYearFortune() 반환값 (현재연도)
  yearFortunes3,      // 3년 세운 배열
  currentDaeun,       // 현재 대운 구간 { gan, ji, age }
  johuInfo,           // getJohuYongsin() 반환값
  // 추가 데이터 (십신/12운성/공망)
  sipsung,            // [년십신, 월십신, 일십신(—), 시십신] — getSipsung(dayP.gan, pillar.gan)
  unsung12,           // [년12운성, 월12운성, 일12운성, 시12운성] — get12Unsung(dayP.gan, pillar.ji)
  gongmang            // getGongmang(dayP.gan, dayP.ji) → { indices: [idx1, idx2], names: [지지1, 지지2] }
};
```

**리스크**: 기존 83개 테스트는 `_lastAnalysisResult`를 참조하지 않음. 유일한 소비자인 `analyzeGunghap()`은 기존 필드만 사용. 새 필드 추가는 기존 동작에 영향 없음.

### 4.2 직렬화 함수 `serializeSajuContext()`

인덱스를 한글/한자로 변환하여 Claude가 명리학 학습 데이터를 직접 활용하도록 한다.

이름: 임재윤 | 성별: 남 | 생년월일: 1994-10-11 | 시: 미시(未時)

■ 사주 원국
        천간    지지    십신    12운성

일주:   庚(경)  午(오)  —      목욕
시주:   癸(계)  未(미)  상관    관대

■ 오행 분포: 목 2.1 | 화 2.2 | 토 5.4 | 금 1.6 | 수 1.0
■ 신강/신약: 신강 | 용신: 수(水) | 희신: 목(木) | 기신: 토(土)
■ 조후용신: 수(水)

  충: (없음)

  2026 丙午: 등급 보통 (편재운)

**설계 원칙**:
- `gan: 0` → `甲(갑)` 변환: Claude의 명리학 학습 데이터 직접 활용

- HTML 태그 제거 (`yearFortune.종합`에 포함된 `<strong>` 등)
- 토큰 효율: JSON 객체 대신 텍스트 테이블 형식으로 ~200 토큰 절약

### 4.3 궁위 매핑 로직

`serializeSajuContext()` 내부에서 합충형파해 항목에 궁위를 자동 부착:

```

년주-일주 → 사회궁-자아궁 (외부 vs 내면)

월주-일주 → 사업궁-자아궁 (직업 vs 나)
월주-시주 → 사업궁-자녀궁 (직업 vs 미래)
일주-시주 → 자아궁-자녀궁 (나 vs 미래/자식)
```

### 5.4 개인화 점수 (핵심 anti-Barnum 테스트)

같은 질문(Q1: "올해 이직하는 게 좋을까요?")을 **3가지 다른 사주**로 호출:

- 사주 A: 기준 케이스 (庚午 일주, 신강, 쌍편재)
- 사주 B: 약한 수 일간 + 식상 과다 (대비 사주)
- 사주 C: 강한 목 일간 + 관살혼잡 (대비 사주)

**판정**: 3개 응답의 핵심 문장이 실질적으로 다른 내용을 담고 있는지 수동 비교. (v2에서 임베딩 유사도 자동화 검토)

### 5.5 거부 테스트

| 입력 | 기대 반응 |
