# 교훈: 구독 모델 전환 + SEO 세션 (2026-03-27)

## 교훈 4: 전략 제안 시 "조사+검증+문서 반영" 세트로 보고

**상황**: 전문가 3인 토론으로 가격 모델(건당→구독) 합의 도출 후, 결과만 사용자에게 보고. 사용자가 "MZ가 사주를 어떻게 쓰는지 조사해야지", "기획에서 모두 정리해야하는 걸 모릅니까?"라고 지적.

**규칙**: 전략적 방향 전환을 제안할 때는 (1) 근거 조사 (2) 기존 계획과의 충돌 검토 (3) 관련 문서 전면 업데이트를 한 세트로 준비한 후 보고한다. 토론 결과만 던지지 않는다.

## 교훈 5: "지금 충돌 아님"은 통하지 않는다

**상황**: 가격 모델이 건당→구독으로 바뀌었는데 "현재 코드(가격표 UI)는 나중에 교체하면 되니까 지금 충돌 아님"이라고 판단. 사용자가 기획 문서 정합성 즉시 반영을 강하게 요구.

**규칙**: 기획 방향이 바뀌면 코드 변경 여부와 관계없이 관련 기획 문서(PRD, TODO, 서비스개요, 기술스택)를 즉시 전부 업데이트한다. 코드는 나중이어도 기획은 즉시.

## 교훈 6: 세션 시작 시 git status 먼저 확인

**상황**: 다른 터미널이 CLAUDE.md를 staged 상태로 남겨뒀는데, 사용자 지적 후에야 발견. staged 변경에 배포 URL 롤백 등 오류 포함.

**규칙**: 세션 시작 시 또는 다른 터미널 작업 병합 시, `git status`로 staged/uncommitted 변경을 먼저 확인하고 내용을 검토한다.
