# 교훈: 수익화 전략 세션 (2026-03-26)

## 교훈 1: 전문가 에이전트 결과는 코드로 검증 후 보고

**상황**: 기술 아키텍트 에이전트가 "getYearFortune() × 12회로 월운 캘린더 가능"이라고 보고. 실제 코드 확인 시 해당 함수는 연단위 전용(dayGanIdx, yearGanIdx, yearJiIdx 파라미터)으로 월별 호출이 불가.

**규칙**: 에이전트가 "기존 함수로 가능하다"고 주장하면, 반드시 함수 시그니처와 호출부를 직접 읽어 확인한 후 사용자에게 보고한다.

## 교훈 2: 코어 문서 업데이트 시 크로스체크 에이전트 필수

**상황**: 3개 코어 문서를 동시에 업데이트했더니 AI 비용이 문서마다 다르게 기재(~100원 vs ~40원), Phase 명칭 혼용 등 7건의 불일치 발생. 별도 리뷰어 에이전트로 전부 잡아냄.

**규칙**: 2개 이상의 문서를 동시 수정할 때는 마지막에 크로스체크 에이전트를 실행하여 가격/Phase/용어 일관성을 확인한다.

## 교훈 3: 사용자가 "재검토해봐"라고 하기 전에 먼저 검증

**상황**: 전문가 합의 결과를 먼저 보고 → 사용자가 "재검토해봐" 요청 → 그제서야 코드 검증. 검증 결과 3건의 오류 발견.

**규칙**: 전문가/에이전트 분석 결과를 사용자에게 보고하기 전에, 핵심 주장 3~5개를 실제 코드/데이터로 검증하는 단계를 반드시 포함한다.
