# whatsyoursaju 해석 품질 전면 개선 — Claude Code 지시 프롬프트

## 맥락

whatsyoursaju(https://whatsyoursaju.vercel.app)는 사주명리 분석 웹 서비스다. 단일 index.html 파일로 구성되어 있고, 계산 엔진(만세력·오행·용신·대운)은 잘 작동하지만, **해석 텍스트 생성 레이어가 얕다.** 현재 일간(10종) × 신강/신약(2종) = 최대 20가지 고정 텍스트 분기만 존재한다.

전통 사주명리학에 기반한 심층 분석서 수준으로 해석 품질을 끌어올리는 것이 목표다. 서버 없이 클라이언트 룰베이스만으로 가능한 범위 내에서 최대한 개선한다.

기준 테스트 케이스: **1994-10-11 14:07 남자** → 甲戌·甲戌·庚午·癸未 (경금 신강, 편인 과다, 쌍편재, 상관생재)

---

## 작업 순서

아래 작업을 **반드시 순서대로** 진행해라. 각 작업이 끝날 때마다 기존 테스트(test_step1~3.js, test_review.js)가 모두 통과하는지 확인해라. 계산 엔진 함수(getSajuYear, getYearPillar, getMonthPillar, getDayPillar, getHourPillar, countElements, getYongsin, getDaeun 등)는 **절대 수정하지 마라.** 해석 텍스트 생성 함수와 UI 렌더링 부분만 수정한다.

### 작업 1: 용신 판단 텍스트 개선

**문제:** getYongsin()은 비겁력 vs 설기력 비교로 용신을 산출하는데, 그 결과를 텍스트에 반영할 때 용신 오행 1개만 표시한다. 실제로는 水·木처럼 복합 용신인 경우가 많다. 또한 getJohuYongsin()이 존재하지만 최종 텍스트에 반영되지 않는다.

**해야 할 것:**
- analyze() 함수에서 이미 호출하고 있는 johuInfo(조후용신)를 결과 HTML의 오행 분석 섹션에 반영해라
- 용신 표시를 "용신: 木(목)" 형태에서 "용신: 水·木 (설기 + 재성 발산)" 형태로 확장해라
- getYongsin()이 리턴하는 용신(1개)에 더해, 희신도 함께 표시해라 (이미 리턴값에 포함되어 있음)
- 조후용신이 억부용신과 다른 경우, "조후용신 참고: ○○" 형태로 별도 노트를 추가해라

(c) 술술(戌戌) 자형(自刑) 감지를 추가해라. findHapChung() 또는 별도 로직으로 년지와 월지가 동일한 지지인 경우(특히 戌戌, 午午, 酉酉, 辰辰)를 감지하고, 성격 분석에 반영해라:

```javascript
// 자형 감지
function detectJahyung(pillars) {
  const jiPairs = [];
  const jiList = pillars.filter(p => p).map(p => p.ji);
  for (let i = 0; i < jiList.length; i++) {
    for (let j = i + 1; j < jiList.length; j++) {
      if (jiList[i] === jiList[j]) {
        const jiName = 지지[jiList[i]];
        // 자형 해당 지지: 辰辰, 午午, 酉酉, 亥亥, 戌戌 (전통적 자형)
        if (['辰','午','酉','亥'].includes(jiName) || jiName === '戌') {
          jiPairs.push(jiName + jiName);
        }
      }
    }
  }
  return jiPairs;
}
```

戌戌 자형이 감지되면 성격 섹션에 다음을 추가: "년지와 월지가 모두 戌(술)으로 자형(自刑)을 이룹니다. '내가 맞다'는 확신이 지나치게 강해질 수 있고, 자기 기준에 미달하면 스스로를 심하게 몰아붙이는 경향이 있습니다. 도움을 요청하는 것을 약점으로 느끼지 말고, 다른 사람의 의견에 귀를 열어두세요."

(d) analyze() 함수의 성격 섹션 HTML 렌더링 부분을 수정해라. 기존 기본+신강/신약 텍스트 아래에, 일주보충 → 구조별보충 → 자형 순서로 텍스트를 이어붙여라. 각 보충 텍스트 앞에 빈 줄(br 2개)을 넣어 가독성을 확보해라.

### 작업 3: 재물운 모순 해결 및 심화

**문제:** getWealthAnalysis()에서 재성 유형을 "편재"로 정확히 감지하면서도, wealthCount 분기(1~2.5)에서 "안정적인 월급 생활에 강점, 적금·연금·부동산 같은 안전 자산" 텍스트가 나온다. 편재형 사주에게 이 조언은 모순이다.

**해야 할 것:**

(a) wealthCount 분기 안에서 wealthType(편재/정재)에 따른 2차 분기를 추가해라:

    text = `재성이 편재 위주로 구성되어, 안정적 월급보다는 <span class="highlight">사업·투자·플랫폼에서 수익을 내는 구조</span>가 체질에 맞습니다. 여러 수입원을 동시에 운영하는 능력이 있지만, 에너지가 분산되지 않도록 핵심 1~2개에 집중하는 것이 중요합니다.`;

  } else {

(b) 상관생재 구조가 감지된 경우(detectSpecialStructures 결과에 '상관생재'가 있을 때), 재물운 텍스트에 다음을 추가해라: "상관생재(傷官生財) 구조를 가지고 있어, 독창적인 아이디어나 서비스를 만들어서 수익화하는 것이 가장 자연스러운 돈벌이 경로입니다. 남이 만든 것을 유통하기보다, 자기가 직접 만든 콘텐츠·서비스·커뮤니티에서 수익을 내는 구조가 사주와 가장 잘 맞습니다."

(c) 쌍편재가 감지된 경우, 재물운에 다음을 추가: "편재가 2개 이상으로, 동시다발적 수입원을 자연스럽게 만드는 구조입니다. 다만 정재(正財)가 없거나 약하면 안정적인 현금 흐름이 구조적으로 부족할 수 있으니, 자동이체 저축이나 비상금 확보 같은 시스템으로 변동성을 완충하세요."

(d) getWealthAnalysis() 함수 시그니처에 specialStructures 파라미터를 추가하고, analyze()에서 detectSpecialStructures()의 결과를 넘겨라.
