## 1. 월별 운세 `getMonthFortune()` 12개월

**적절한 시점**: Phase 2 (결제 시스템 구축 이후)
**참조**: TODO.md Phase 2-3 (월운 캘린더)

**설계 메모**:
- `getYearFortune()` 확장이 아닌 **신규 함수** 필요 (getYearFortune은 연도 단위 전용)
- `getMonthPillar(targetYear, month, 15)` 로 각 월의 월주 간지 계산
- 일간과 월주 천간/지지의 십신 관계 기반 월별 등급 + 조언 텍스트 생성
- **무료/유료 경계**: 현재 월 + 다음 달만 무료, 나머지 10개월 블러 → 가장 자연스러운 프리미엄 전환 트리거
- 올해 탭(~4170줄)에 월별 캘린더 그리드 UI 삽입

**제외 이유**: 결제 시스템이 없는 상태에서 블러 처리해도 수익으로 이어지지 않음. TODO.md Phase 2 계획과 중복.

## 2. 각 탭별 블러 프리미엄 카드 (4개 추가)

**적절한 시점**: Phase 1 이후 (유료 상품 구현 후)

**설계 메모**:
- 올해 탭: "월별 행운 캘린더" 블러 (월별 운세와 연계)
- 사랑 탭: "2026년 연애 타이밍 분석" 블러
- 재물 탭: "재물 리스크 5대 포인트" 블러
- 건강 탭: "오행별 체질 맞춤 식단표" 블러
- 기존 `.premium-card-blur` CSS 클래스 재사용 가능

**제외 이유**: 현재 이미 5개 전환 터치포인트(인라인 티저 3개 + 블러 카드 1개 + 충격의 한 줄 1개) 존재. 과하면 "돈 내라" 스팸 느낌 → 무료의 관대함(바이럴 원동력) 훼손.

## 3. 프리미엄 가격표 재정의

**적절한 시점**: Phase 0.5 가격 민감도 조사 이후

**설계 메모**:
- 현재 3단(인사이트 9,900 / 마스터 19,900 / VIP 39,900)
- 무료에서 이미 제공하는 "재물·직업 분석"이 가격표에 차별점으로 나와 있어 모순
- 프리미엄 핵심 가치를 "AI 상담 + 월별 캘린더 + 심층 궁합"으로 재구성 필요
- TODO.md Phase 1-5 가격표 업데이트 참조

**제외 이유**: AI 상담 상품 미구현, 가격 미확정 (1,900 vs 3,900 vs 5,900 테스트 예정).

## 4. 영역별 "충격의 한 줄" 복수 생성

**적절한 시점**: GA 트래픽 데이터 축적 후

**설계 메모**:
- 현재 `generateTeaserInsight()` (3674줄)가 전체에서 1개 선택
- 재물/사랑/건강 탭 상단에 각 영역 전용 훅 1줄씩 배치
- 영역별 인사이트: 배우자궁 12운성, 편재 개수, 오행 과다/부족 등 활용

**제외 이유**: 강력한 1개 인사이트 > 약한 여러 개. GA에서 탭별 이탈률 확인 후 필요한 탭에만 선별 배치하는 게 데이터 기반 의사결정.

## 5. 궁합 미니게임 → 사랑탭 이동

**적절한 시점**: A/B 테스트 인프라 구축 후

**설계 메모**:
- 현재: Tier 3 하단 고정 → 모든 사용자에게 노출
- 사랑 탭 내부 이동 시: 연애운→궁합의 자연스러운 흐름, BUT 사랑 탭 미클릭 사용자는 궁합을 못 봄
- GA `gunghap_start` 이벤트 기준으로 현재 도달률 먼저 확인 필요

**제외 이유**: 데이터 없이 위치 변경은 위험. 현재 위치(전체 노출)가 바이럴에 유리할 가능성 높음.
