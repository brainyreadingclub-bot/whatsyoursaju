# PRD — SAJU (What's Your Saju)

> 이 문서는 AI 에이전트가 이 서비스를 구현할 때 참조할 "판단 기준"이다.
> 구체적인 코드 구현 방법(How)이 아닌, 무엇을(What) 왜(Why) 만드는지에 대한 맥락을 담는다.

---

## 1. 배경 및 맥락

### 왜 이 서비스가 필요한가

- 한국 점술 시장은 1.4조~4조원 규모이며, MZ세대의 80%가 사주/운세를 이용한다
- 기존 서비스는 **개인 1:1 분석**에 그친다 — 관계 맥락(커플/팀/가족) 분석은 시장에 부재
- 무료 서비스는 만세력 계산기 수준, 유료 서비스는 전화상담(점신) 또는 캐릭터 운세(포스텔러)
- **"나와 주변 사람들의 관계를 체계적으로 분석"하는 AI 기반 서비스**가 새로운 카테고리를 만든다

### 핵심 인사이트

- 사주 분석의 가치는 **계산의 정확성** + **해석의 깊이** + **처방의 구체성**에 있다
- 사주는 '계산'이 아니라 '읽기(讀命)'다 — 같은 사주도 해석이 다르면 가치가 다르다
- **제품 가치의 70%는 AI 해석 품질(프롬프트 엔지니어링)에 달려있다**
- 무료 분석의 풍부함이 신뢰를 만들고, 신뢰가 유료 전환을 만든다
- 서버 없이 동작하는 무료 기능 + 서버리스 유료 기능 = 최소 비용 최대 수익 구조

## 2. 목적 및 목표

### 해결하려는 문제

> 나를 알고, 내 관계를 읽고, 시기별 처방까지 받을 수 있는 **사주 기반 인텔리전스 서비스**의 부재

### 목표

| 목표 | 기준 |
|------|------|
| 분석 정확도 | 전문 만세력 기준 4주(년·월·일·시) 간지 100% 일치 |
| 사용자 경험 | 입력 → 결과까지 3단계(이름·생년월일시·성별) 이내 |
| 응답 속도 | 무료 분석 2초 이내, AI 상담 응답 5초 이내 |
| 접근성 | 모바일/데스크톱 브라우저에서 가입·설치 없이 즉시 사용 |
| 전환율 | 무료→AI상담 전환 5~8% |
| 수익 | Phase 2 종료 시 MRR 500만원 |

## 3. 주요 사용자 스토리

### 무료 사용자 (진입 퍼널)

1. **"호기심 사용자"로서**, 나는 **이름과 생년월일시만 입력**하고 싶다. 그래서 **복잡한 사주명리 계산 결과를 이해하기 쉬운 언어로** 즉시 받아볼 수 있다.

2. **"자기이해를 원하는 사용자"로서**, 나는 **성격·적성·강점·약점에 대한 깊이 있는 분석**을 받고 싶다. 그래서 **사주라는 렌즈를 통해 나 자신을 다른 관점에서** 이해할 수 있다.

3. **"공유를 원하는 사용자"로서**, 나는 **결과를 예쁜 이미지 카드로 저장**하고 싶다. 그래서 **친구나 SNS에 공유하며 대화 소재**로 활용할 수 있다.

4. **"올해 운세가 궁금한 사용자"로서**, 나는 **현재 연도 기준의 운세**를 확인하고 싶다. 그래서 **올해 주의할 점과 기회를 파악**할 수 있다.

5. **"궁합이 궁금한 사용자"로서**, 나는 **상대방 생년월일만 넣어** 간단한 궁합 점수와 케미를 확인하고 싶다.

6. **"시간을 모르는 사용자"로서**, 나는 **태어난 시간 없이도 분석을 진행**하고 싶다.

### 전환 사용자 (유료 진입)

7. **"AI 상담을 원하는 사용자"로서**, 나는 기본 분석을 본 뒤 **AI에게 구체적 상황을 설명하고 맞춤 답변**을 받고 싶다. ("올해 이직해야 할까?", "이 사람과 결혼해도 될까?")

8. **"프리미엄 궁합을 원하는 사용자"로서**, 나는 미니게임 점수를 넘어 **왜 이 궁합인지, 갈등 포인트가 뭔지, 어떻게 조화를 이룰지** 깊은 분석을 원한다.

### 프리미엄 사용자

9. **"그룹 분석을 원하는 사용자"로서**, 나는 **우리 팀/가족 전원의 사주를 종합 분석**해서 역할 분화, 시너지, 갈등 구조, 10년 타임라인을 확인하고 싶다.

10. **"재방문 사용자"로서**, 나는 대운 전환기나 새해에 다시 찾아와 **새로운 세운에 맞는 상담**을 받고 싶다.

## 7. 전환 퍼널 설계

```
무료 분석 (100%)
  → 결과 페이지에서 AI 맛보기 노출 (40% 클릭 예상)
    → AI 맛보기 1회 무료 — 이메일 인증 (30% 완료)
      → "더 깊은 답변" 유료 전환 (5~8%)
        → 세션 업셀 (전환자의 40%)
          → 궁합/그룹 크로스셀
```

### 전환 원칙
- 감정 최고조(충격의 한 줄, AI 맛보기 응답) 직후에 자연스러운 업셀
- 강요가 아닌 **"더 알고 싶다"는 욕구의 연장**
- 무료 경험이 충분히 가치 있어야 유료 전환이 자연스러움

## 8. 분석 품질 기준

| 계층 | 기대 수준 | 벤치마크 |
|------|-----------|----------|
| **무료** | 교과서적 해석, 용어 순화, 일상 언어 | 현재 index.html 풀이 텍스트 수준 |
| **AI 상담** | 개인 맥락 반영 해석+시기별 처방 | "당신의 경금 일간에 올해 병오년은 식신이 강한 해이므로..." 수준 |
| **프리미엄 궁합** | 합충 근거 + 갈등 원인 + 관계 처방 | 천간합/지지충의 실생활 의미까지 해석 |
| **그룹 분석** | MECE 프레임워크 수준 — 개인→쌍대→그룹→시간축→실행 | 파운더 MECE 종합분석서 참조 |

### 사주 AI 프롬프트 요구사항

시스템 프롬프트에 명리학 해석 프레임워크를 내장해야 한다:

- **일간 강약 판단** → 용신 활용법 → 기신 회피법
- **합충형파해의 실생활 적용** — 관계/직업/건강에 미치는 구체적 영향
- **대운/세운 교차 해석** → 시기별 처방 ("이 시기에는 ○○하세요")
- **궁합 해석** — 천간합/지지충의 관계 의미, 갈등 구조, 중재 방안
- **그룹 역학** — 을경합 등 팀 내 합의 의미, 오행 편중의 보완 전략

품질 원칙: **"계산 결과를 그대로 읽어주는 것이 아니라, 맥락을 읽어 처방하는 것"**

## 9. UX 및 디자인 가이드

### 디자인 원칙

| 원칙 | 구현 방향 |
|------|-----------|
| **신비로운 톤** | 우주/별자리 느낌의 다크 배경(#0a0610), 금색+보라색 그라디언트 |
| **정보 밀도 조절** | 핵심 결과는 크게, 상세 풀이는 카드/탭 단위로 분리 |
| **한자+한글 병기** | 전통성(한자)과 접근성(한글)의 균형 — "甲戌(갑술)" 형태 |
| **오행 색상 체계** | 목(#4a9e6b), 화(#d4534a), 토(#c4953a), 금(#b8b8cc), 수(#4a7ec4) |
| **모바일 퍼스트** | 세로 스크롤 중심, 터치 영역 충분, 가로 오버플로 없음 |

### 결과 페이지 구조

```
[Tier 1] 사주테이블 + 합충형파해 + 오행 + 일간요약 + 공유CTA
[충격의 한 줄] 전환 트리거 (호기심 유발)
[Tier 2] 탭 UI: 올해 | 사랑 | 재물 | 성격 | 건강 | 심화
[AI 맛보기] "이 사주에 대해 더 궁금한 점을 물어보세요" (Phase 1)
[궁합 미니게임] 바이럴 엔진
[Tier 3] 프리미엄 블러 카드 + 가격표
```

## 11. 릴리즈 계획

### Phase 0.5: 검증 스프린트 (2주)

- Formspree 이메일 리스트로 베타 초대 (10~20명)
- AI 단일 질문 상담 품질 테스트
- 프롬프트 튜닝 + 사용자 피드백 수집
- 가격 민감도 조사 (1,900 vs 3,900 vs 5,900원)
- **검증 기준**: AI 응답에 "와, 이걸 어떻게 알지?" 반응이 나오는가

### Phase 1: 전환 기반 구축 (+2개월)

- Vercel Functions 도입 (/api/*)

## 12. 가정, 제약 사항 및 의존성

### 가정 사항

- 대부분의 사용자는 **양력 생년월일**을 알고 있다 (음력은 변환 기능 제공)
- 사용자는 **모바일 환경**에서 주로 접근한다
- AI 상담의 품질은 **프롬프트 엔지니어링 수준에 비례**한다
- 무료 사용자의 5~8%가 유료 전환할 의향이 있다

### 제약 사항

- **절기일 평균값 한계**: ±1일 오차 가능
- **단일 파일 구조**: index.html 4,850줄. Phase 2에서 saju-engine.js 분리 계획
- **AI 비용**: Claude API 종량제. 일 100건 기준 월 ~$30
- **사업자 등록**: 실제 결제 서비스에는 사업자등록 필요 (PortOne 테스트 모드는 가능)

### 의존성

| 의존성 | 영향 |
|--------|------|
| Google Fonts CDN | 폰트 로드 실패 시 시스템 폰트 폴백 |
| html2canvas CDN | 이미지 카드 생성 불가 시 공유 기능 제한 |
| korean-lunar-calendar CDN | 음력 변환 불가 시 양력만 지원 |
| Vercel | 배포 + 서버리스 함수 + KV |
| Claude API (Anthropic) | AI 상담 핵심 의존성. 장애 시 유료 기능 비가용 |
| PortOne | 결제 처리. 장애 시 결제 불가 (무료 기능 영향 없음) |
