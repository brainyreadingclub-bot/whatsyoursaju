# PRD — SAJU (What's Your Saju)

> 이 문서는 AI 에이전트가 이 서비스를 구현할 때 참조할 "판단 기준"이다.
> 구체적인 코드 구현 방법(How)이 아닌, 무엇을(What) 왜(Why) 만드는지에 대한 맥락을 담는다.

---

## 1. 배경 및 맥락

### 왜 이 서비스가 필요한가

- 사주명리학은 한국에서 문화적으로 깊이 뿌리내린 자기이해/운세 체계이지만, 온라인에서 **정밀하면서도 무료인** 분석 서비스가 부족하다
- 기존 무료 사주 서비스는 단순 간지 조회 수준이거나, 깊이 있는 분석은 유료로 전환을 유도한다
- 새해/생일/전환기에 "내 사주" 검색 수요는 꾸준히 존재하며, 바이럴 가능성이 높은 콘텐츠 유형이다

### 핵심 인사이트

- 사주 분석의 핵심 가치는 **계산의 정확성**과 **풀이의 공감력**이다
- 사용자는 "맞는 것 같다"는 느낌을 받아야 다시 방문하고, 공유한다
- 서버 없이 브라우저만으로 동작하는 구조는 운영 비용 0원과 즉시 응답이라는 압도적 장점을 준다

---

## 2. 목적 및 목표

### 해결하려는 문제

> 사주명리에 관심 있는 사용자가 **믿을 수 있는 수준의 사주 분석**을 **무료로, 가입 없이, 즉시** 받아볼 수 없다

### 목표

| 목표 | 기준 |
|------|------|
| 분석 정확도 | 전문 만세력 기준 4주(년·월·일·시) 간지 100% 일치 |
| 사용자 경험 | 입력 → 결과까지 3단계(이름·생년월일시·성별) 이내 |
| 응답 속도 | 분석 결과 생성까지 2초 이내 (로딩 애니메이션 포함) |
| 접근성 | 모바일/데스크톱 브라우저에서 가입·설치 없이 즉시 사용 |
| 공유성 | 결과 이미지 카드 생성 → SNS 공유 1탭 완료 |

## 3. 주요 사용자 스토리

### 핵심 스토리

1. **"호기심 사용자"로서**, 나는 **이름과 생년월일시만 입력**하고 싶다. 그래서 **복잡한 사주명리 계산 결과를 이해하기 쉬운 언어로** 즉시 받아볼 수 있다.

2. **"자기이해를 원하는 사용자"로서**, 나는 **성격·적성·강점·약점에 대한 깊이 있는 분석**을 받고 싶다. 그래서 **사주라는 렌즈를 통해 나 자신을 다른 관점에서** 이해할 수 있다.

3. **"공유를 원하는 사용자"로서**, 나는 **결과를 예쁜 이미지 카드로 저장**하고 싶다. 그래서 **친구나 SNS에 공유하며 대화 소재**로 활용할 수 있다.

4. **"올해 운세가 궁금한 사용자"로서**, 나는 **현재 연도 기준의 운세**를 확인하고 싶다. 그래서 **올해 주의할 점과 기회를 파악**할 수 있다.

5. **"관계를 알고 싶은 사용자"로서**, 나는 **나와 가까운 사람들(최대 5명)의 사주를 함께 입력**하고 싶다. 그래서 **서로의 오행이 어떻게 보완되고, 누가 어떤 역할에 맞는지, 장단점이 어떻게 채워지는지** 한눈에 볼 수 있다.

6. **"그룹 케미가 궁금한 사용자"로서**, 나는 **우리 그룹의 전체 오행 균형과 시너지**를 확인하고 싶다. 그래서 **팀/커플/가족으로서 어떤 강점과 약점이 있는지** 파악하고 대화 소재로 활용할 수 있다.

### 부가 스토리

7. **"시간을 모르는 사용자"로서**, 나는 **태어난 시간 없이도 분석을 진행**하고 싶다. 그래서 시주 제외 3주 기반의 분석이라도 받을 수 있다.

## 4. 성공 지표

### 정량적 지표

| 지표 | 기준 |
|------|------|
| 4주 간지 정확도 | 만세력 기준 100% 일치 (test_step1~3 전체 통과) |
| 분석 완료율 | 입력 시작 → 결과 확인까지 이탈 없이 완료하는 비율 |
| 공유 전환율 | 결과 페이지에서 이미지 저장 또는 URL 공유를 실행하는 비율 |
| 모바일 사용성 | 모바일 기기에서 입력~결과까지 스크롤/줌 없이 완료 가능 |

### 정성적 지표

| 지표 | 기준 |
|------|------|
| 분석 공감도 | "맞는 것 같다"는 반응을 유도할 수 있는 풀이 텍스트의 구체성과 개인화 |
| 시각적 완성도 | 전문 서비스 수준의 비주얼 품질 (우주/신비로운 톤) |
| 재방문 의향 | "다른 사람 것도 봐야지" → 추가 사용 욕구 유발 |

## 5. 제품 개념 및 주요 기능

### 핵심 기능 (반드시 포함)

| 기능 | 설명 |
|------|------|
| **사주팔자 계산** | 입춘 기준 연도, 절기 기준 월주, 일주·시주 정밀 계산 |
| **오행 분석** | 천간+지지+장간 가중치 반영 오행 분포 산출 및 시각화 |
| **용신 판단** | 신강/신약 기반 용신·희신·기신 산출 + 조후용신 |
| **종합 풀이** | 성격, 재물운, 연애운, 건강, 직업운을 일상 언어로 풀이 |
| **대운 흐름** | 순행/역행 대운 8개 + 시작나이 자동 산출 |
| **올해 운세** | 현재 연도 기준 세운 분석 (분기별 포인트 포함) |
| **결과 공유** | 이미지 카드 생성 + URL 복사 + Web Share API |
| **그룹 궁합 분석** | 최대 5인의 사주를 입력하여 서로의 오행 보완 관계, 장단점 매칭, 그룹 내 역할과 시너지 분석 |

### 이번 릴리즈에서 제외되는 항목

| 제외 항목 | 이유 |
|-----------|------|
| **음력→양력 변환** | 정밀한 천문 데이터 테이블 필요, 부정확한 변환보다 미지원이 나음 |
| **회원 가입/로그인** | 서버 없는 구조의 장점을 유지하기 위해 제외 |
| **분석 이력 저장** | 서버 없이는 LocalStorage만 가능, 우선순위 낮음 |
| **AI 실시간 상담** | LLM API 비용 발생, 규칙 기반 엔진의 신뢰성 우선 |
| **정밀 절기 데이터** | 연도별 실제 절기일 테이블 구축 필요, 현재 평균값으로 충분 |

## 6. UX 및 디자인 가이드

### 핵심 사용자 흐름

```
[개인 분석]
[랜딩] → [이름·성별·생년월일시 입력] → [분석 시작] → [로딩] → [결과] → [공유]

[그룹 궁합]
[랜딩] → [모드 선택: 개인 / 그룹] → [1인씩 정보 입력 (최대 5인)]
    → [궁합 분석 시작] → [로딩] → [그룹 결과 페이지]
    → [오행 보완 관계 / 장단점 매칭 / 역할 분석 / 공유]
```

### 디자인 원칙

| 원칙 | 구현 방향 |
|------|-----------|
| **신비로운 톤** | 우주/별자리 느낌의 다크 배경(#0a0610), 금색+보라색 그라디언트 |
| **정보 밀도 조절** | 핵심 결과(4주 간지)는 크게, 상세 풀이는 카드 단위로 분리 |
| **한자+한글 병기** | 전통성(한자)과 접근성(한글)의 균형 — "甲戌(갑술)" 형태 |
| **오행 색상 체계** | 목(#4a9e6b), 화(#d4534a), 토(#c4953a), 금(#b8b8cc), 수(#4a7ec4) |
| **모바일 퍼스트** | 세로 스크롤 중심, 터치 영역 충분, 가로 오버플로 없음 |

### 사용자가 결과 페이지에서 보는 순서

1. 사주팔자 4주 카드 (한자 크게 + 오행 색상)
2. 대운 흐름 타임라인
3. 일주 12운성
4. 오행 분석 차트 + 과다/부족
5. 성격 및 기질 (일간별 신강/신약 분기)
6. 재물운
7. 직업운 및 적성
8. 연애운
9. 건강 분석
10. 올해 운세 (분기별)
11. 행운 아이템 + 피해야 할 것/가까이할 것
12. 공유 카드

## 7. 시스템 요구 사항

| 요소 | 선택 | 이유 |
|------|------|------|
| **아키텍처** | 단일 HTML 파일 (프론트엔드 전용) | 서버 비용 0원, 배포 단순, 즉시 응답 |
| **배포** | Vercel 정적 호스팅 | 무료, 전역 CDN, 자동 HTTPS |
| **계산 엔진** | 순수 JavaScript (인라인) | 외부 의존성 최소화, 오프라인에서도 동작 |
| **이미지 생성** | html2canvas (CDN) | 클라이언트 사이드 스크린샷, 서버 불필요 |
| **폰트** | Google Fonts (Noto Serif KR 등) | 동양적 서체, 무료, CDN |
| **지원 브라우저** | Chrome, Safari, Edge 최신 2개 버전 | 모던 CSS/JS 활용, IE 미지원 |

## 8. 가정, 제약 사항 및 의존성

### 가정 사항

- 대부분의 사용자는 **양력 생년월일**을 알고 있다 (음력만 아는 경우는 외부 변환 후 입력)
- 사용자는 **모바일 환경**에서 주로 접근한다
- 사주 분석의 "정확도"에 대한 기준은 **전통 만세력 4주 간지 일치 여부**로 판단한다
- 풀이 텍스트의 "적절함"은 **바넘 효과를 넘어서는 구체적 맥락 제공**으로 판단한다

### 제약 사항

- **절기일 평균값 한계**: 절기일은 연도별로 ±1일 차이가 있으나, 현재 고정 평균값 사용. 절기 당일 출생자는 오차 가능
- **음력 미지원**: 정밀한 음력→양력 변환 테이블(천문 데이터) 없이는 오변환 위험이 있어 의도적으로 차단
- **서버 없음**: 사용자 데이터 저장, 분석 이력, A/B 테스트 등 서버 의존 기능은 현재 구조에서 불가
- **단일 파일 구조**: 현재 index.html 2,260줄로 유지보수성 한계가 있음. 기능 추가 시 모듈 분리 필요

### 의존성

| 의존성 | 영향 |
|--------|------|
| Google Fonts CDN | 폰트 로드 실패 시 시스템 폰트 폴백 |
| html2canvas CDN | 이미지 카드 생성 불가 시 공유 기능 제한 |
| Vercel | 배포 플랫폼 장애 시 서비스 접근 불가 |
| 브라우저 JavaScript | JS 비활성화 시 서비스 완전 비동작 |

## 9. 릴리즈 계획

### Phase 1: MVP + 그룹 궁합 (현재 → 다음 릴리즈)

- 4주 계산 정확도 100% 검증 (test_step1~3 전체 통과)
- **그룹 궁합/케미 분석 기능 구현 (최대 5인)**
  - 멀티 유저 입력 UI
  - 오행 보완 분석, 장단점 매칭, 그룹 역할 분석
  - 그룹 결과 공유 카드
- 모바일/데스크톱 크로스 브라우저 테스트

### Phase 2: 기능 확장

- 음력→양력 변환 지원
- 연도별 정밀 절기 데이터 적용
- 일진(오늘의 운세) 기능
- SEO 메타태그 최적화 + OG 이미지

### Phase 3: 구조 개선

- index.html → 모듈 분리 (계산 엔진 / UI / 데이터)
- 테스트 코드와 본문 로직 통합 (중복 제거)
- 성능 최적화 (코드 스플리팅, 레이지 로딩)

### 검증 계획

- 기존 test_step1~3.js를 활용한 회귀 테스트
- 다양한 출생년도·시간 조합으로 만세력 교차 검증
- 실 사용자 피드백 수집 (SNS 공유 후 반응 모니터링)
