## 기술적 주의사항 (세션 간 인계용)

### 코드 수정 시 동기화 필수

현재 계산 함수가 3곳에 복사되어 있다 (Phase 2에서 해결 예정):
1. `index.html` (line 2356-2620) — 실서비스
2. `test_step1~3.js`, `test_review.js` — 테스트
3. `/api/lib/saju-engine.js` (Phase 1에서 생성) — 서버리스

**한쪽 수정 시 나머지도 반드시 동기화한다.**

### window._lastAnalysisResult 확장 필요

현재 저장 필드 (line 4010):
```
{ name, dayP, yearP, monthP, hourP, pillars, elementCounts, yongsinInfo }
```

AI 상담에 필요하지만 현재 미저장:
```
specialStructures, sinsalList, hapChung, daeunList, yearFortune
```
→ Phase 1에서 analyze() 함수 내 저장 로직 확장 필요

### getYearFortune()은 월별 호출 불가

이전 세션에서 "getYearFortune() × 12 = 월운 캘린더" 주장은 **검증 결과 오류**였다.
- getYearFortune()은 targetYear(연도)를 파라미터로 받아 세운 천간/지지를 계산
- 월별 호출 시 필요한 월주(月柱) 계산 로직이 함수 내부에 없음
- Phase 2에서 `getMonthFortune()` 신규 함수 개발 필요

### 결제 보안 핵심 원칙

- 금액은 **반드시 서버 상수에서 조회** (클라이언트 전달 금액 무시)
- 결제 검증: PortOne API로 실제 결제 금액 조회 → 서버 금액과 비교
- API 키: 서버리스 함수 환경변수에만 저장 (절대 클라이언트 노출 금지)
- 토큰: 서버 발급 JWT/UUID, Vercel KV에서 검증
